# DIKWP-CIVICLOOP²

**实质性人类控制、算法决策可争议、伦理规则演化与普惠收益公共运行时**

DIKWP-CIVICLOOP² is an offline-first reference runtime for compiling four public-AI questions into action-specific evidence:

1. How do humans and increasingly agentic AI coexist?
2. How is safety protected when algorithms participate in decisions?
3. How can ethical and governance rules evolve without becoming an unaccountable fixed ontology?
4. How can access, capability, outcome, voice and value capture be audited rather than assumed under the label of inclusion?

The runtime **does not define** “human control”, “ethics”, “safety” or “inclusion” once and for all. It evaluates each action in a concrete observer, domain, time, authority and consequence context. D/I/K/W/P are co-equal semantic resources; all 25 ordered DIKWP×DIKWP transformations are first-class traces.

## What is included

- Actor and agent passports
- Definition-free intent fields
- Decision-rights graph and action modes
- Meaningful Human Control Proof
- Algorithmic Decision Receipt
- Appeal and Repair Ticket
- Ethical Rule Patch with expiry and rollback
- Inclusion and Benefit Ledger
- Five China-oriented policy profiles
- Append-only SHA-256 decision ledger
- 180 deterministic synthetic events across nine domains
- Offline HTML dashboard and automated tests

## Quick start

```bash
cd DIKWP_CIVICLOOP2_MVP
PYTHONPATH=src python -m dikwp_civicloop --root .
PYTHONPATH=src python -m unittest discover -s tests -v
```

Open `outputs/dashboard.html` in any modern browser.

## Strict boundaries

This package is not a legal decision system, clinical system, employment model, benefit eligibility engine, autonomous industrial controller, or official compliance certification. It must not be connected to real rights, payments, medical care, education opportunities, public benefits, identity systems or physical equipment without lawful authority, domain validation, security testing, affected-party participation, meaningful appeal, recovery exercises and independent settlement.

The demo data is synthetic. A high score or `PROVEN` control state in this reference implementation does not certify any real deployment.
